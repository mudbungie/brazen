Arch-review finding (2026-07-07), the composed P0: each piece is individually spec-owned, but together they break the marquee use-case — a --json harness running an agentic tool loop with reasoning enabled.

THE CHAIN, per dialect:
- Anthropic: decode drops signature_delta (src/protocol/anthropic/decode/blocks.rs:63 -> vec![]; anthropic-messages.md CR-5 'low urgency') and redacted_thinking's data is never captured (blocks.rs:33 opens ContentKind::RedactedThinking{} empty; spec line ~731). So the harness can only build Thinking{signature:None}, which encode then DROPS (encode/blocks.rs:42-44, CR-2 rule). Anthropic REQUIRES the prior assistant turn's thinking blocks with signature when continuing after tool_use with thinking enabled -> the replay 400s ('Expected thinking or redacted_thinking, but found tool_use') or thinking must be disabled. CR-5's 'low urgency' is a misconception: reasoning replay IS the agent loop.
- Google: thoughtSignature on parts (notably functionCall parts, Gemini 2.5) is neither captured on decode (google_genai/decode/blocks.rs:21-46 reads only text/thought/functionCall) nor emitted on encode (encode/contents.rs:59-63 drops Thinking). Multi-turn function calling on Gemini thinking models degrades/breaks.
- OpenAI Responses: reasoning items decode to empty ContentKind::Thinking{} with no item id / encrypted_content capture (openai_responses/decode/mod.rs:85); providers.md CR-R3 defers replay as 'low urgency' — but the codex/ChatGPT-SSO row MANDATES store:false, where o-series/gpt-5 multi-turn tool loops need encrypted reasoning item round-trip (include: reasoning.encrypted_content) or the model loses its chain between calls.

DESIGN-FIRST (spec before impl). The canonical decision is ONE design: how opaque replay payloads ride the event vocabulary. All additions are additive under the v=1 grows-only contract (architecture.md §3.2/§13.9 — new delta/kind/field never bumps v). Candidate shape (argue in spec): Delta::SignatureDelta(String) (wire arrival order preserved; sink folds onto Content::Thinking.signature — CR-5's own sketch); ContentKind::RedactedThinking gains {data} at ContentStart (arrives at block open — mirror ServerToolResult's inline-content precedent); Responses Thinking kind gains optional {id, encrypted_content} (encrypted_content arrives on output_item.done — decide delta-vs-stop carriage); Google thoughtSignature analogously (it rides the part — may belong on ToolUse kind too, since it attaches to functionCall parts; check wire truth first). Request-side Content may need matching optional fields (Content::ToolUse gaining an optional signature for Gemini? verify before inventing). Also fix decode_full paths to carry the same facts.

Specs: architecture.md §3.1/§3.2 first (parent), then anthropic-messages.md (§3.4, CR-2/CR-5 resolution), providers.md (google, CR-R3 resolution), event fixtures. Consider splitting impl per dialect into subtasks after the spec lands — the vocabulary decision must be one ball.
