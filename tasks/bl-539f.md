release-plz pushes a new timestamped `release-plz-*` branch every time it
refreshes the release PR, and it pushes the branch BEFORE opening the PR.

brazen accumulated 25 such branches against only one ever-created PR — 24 were
orphaned because PR creation was 403ing until "Allow GitHub Actions to create
and approve pull requests" was enabled on the repo. Those 24 have now been
deleted by hand, and "Automatically delete head branches" has been enabled on
the repo, but that setting only fires on MERGE — it does nothing for closed or
orphaned branches.

Fix: append a `prune-release-branches` job to
`.github/workflows/release-plz.yml`. It `needs: release-plz-pr`, so it runs
right after the release-PR job — the branch backing the current release PR is
already open, making "has no open PR" an exact test for "superseded or
orphaned". No allowlist, no age heuristic, no stored state: the open-PR set is
the single source of truth.
