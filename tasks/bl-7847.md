Arch-review finding (2026-07-07), two holes in the stream-shape contract a harness leans on:

1) On error termination (premature EOF, mid-stream transport drop) the run loop injects Error then End WITHOUT closing open content blocks (run/events/stream.rs:71-89): the consumer sees ContentStart/ContentDelta... Error, End with no ContentStop. The 'every delta bracketed by start/stop' invariant is asserted only for clean fixtures (src/tests/protocol_sse_determinism.rs:184-198, guarded on clean streams) and is NOT promised either way in architecture.md §5.6. An embedder finalizing per-block state on ContentStop leaks/hangs on every truncated stream. Decide + spec: the clean invariant is 'run drains DecodeState.open and emits ContentStop for each open index before the injected Error' — every start eventually stops, even on failure (the same dissolve-the-edge-case move as the unconditional End). Implement in the run loop (decode stays pure), extend the determinism invariant to error streams.

2) decode_full has no completeness check (run/events/mod.rs:50-57): an empty/malformed non-stream 200 body ({} or {"choices":[]}) flows through choices[0]-Null tolerance to yield MessageStart + End only — no Finish, no Error, exit 0. A consumer sees a silently-empty successful turn. Decide + spec (architecture.md §3.2 non-stream fold): a decode_full yielding no Finish and no Error is a malformed aggregate -> in-band Error (ParseInput-or-Transport, argue which) instead of silent success. Fixture per dialect for the empty aggregate.

Both are additive contract strengthenings (no v bump: adding events on failure paths grows the vocabulary's guarantees, removes none). architecture.md §5.6 wording, CHANGELOG.
