Dedup the protocol layer along the grain of what is *actually* shared vs. what
merely looks shared. The line-split refactor (bl-78f7/bl-ec6b/bl-d283) reorganized
each protocol `decode` into mod/blocks/errors but did NOT dedup across providers.

**Single source of truth.** The canonical model (`CanonicalRequest`/`Event`) is
already the consolidated core; every protocol is a lossy projection onto it.
Consolidate ONLY mechanics the canonical model *implies* (drain order, index
discipline, JSON access) — never the wire shapes, which ARE the divergence. A shared
engine that eats wire-shape variation becomes a second source of truth that drifts.

**Two families.**
- Explicit-structure (anthropic, openai_responses): wire carries block start/stop.
- Synthesized-structure (openai_chat, google, ollama): no block markers — synthesize
  MessageStart/ContentStart, lazy text block, monotonic `open.len()` index, drain
  all-open at the terminal. google and ollama `blocks.rs` came out nearly verbatim.

**Deliverable 1 — unconditional, cheap.** Extract the leaf JSON accessors
(`text_of`, `nonempty`, `next_index`/`index`, `to_json_string`, `u32_at`) into a
shared `src/protocol/json.rs` (or a `JsonExt` trait). Zero divergence risk; replaces
the per-provider copies currently `pub(super)` in each decode `mod.rs`.

**Deliverable 2 — design-first (spec), then conditional impl.** A shared
"synthesized-stream decoder" skeleton + per-provider extractors (read model id /
content parts / finish reason / usage / error envelope). Skeleton owns the
drain/index/ordering discipline; providers supply small pure fns.
  - GO/NO-GO GATE: fit `openai_chat` into the skeleton ON PAPER first. openai is a
    cousin, not a twin — it streams text AND tool-args as fragments (google/ollama
    deliver tool calls whole) and uses a `tool_index` map (others use the bare index).
    If those land as small extractor fns -> build it (kills the biggest duplication).
    If they become bool flags (`tool_args_are_whole`, `text_is_fragmented`) -> ABORT:
    the abstraction is leaking into a config-language; keep providers independent and
    ship only Deliverable 1.

**Scope guard.** Leave `encode` and the explicit-structure family independent —
encode bodies are genuinely per-wire (keys/nesting/renames diverge), so there is no
real skeleton beyond the leaf helpers + the text-only-slot/`slot_err` pattern.

**Workflow.** Design first: the design-phase deliverable is a tracked spec (e.g.
specs/00NN-protocol-dedup.md), edited like code; implementation follows, gated by the
fit-test above. Standard close gate applies (100% coverage, <=300 lines, clippy -D).
