bl-2cee (e825817) wired auto-push as .githooks/post-commit, but bl close advances main via a compare-and-swap ref move (bl close --skill: 'the delivery ref move is a compare-and-swap'), not git commit — post-commit never fires on delivery, which is the primary (and per workflow rules, only) way commits land on main. Observed: neither the bl-2cee nor bl-09c6 close auto-pushed. Fix: replace post-commit with a versioned .githooks/reference-transaction hook — on state 'committed', if a stdin line moved refs/heads/main (old != new; exact-ref match so remote-tracking updates from the push itself cannot recurse), git push origin main, non-fatal warn on failure. Delete .githooks/post-commit. Update AGENTS.md close-gate #2 wording. Local clones chaining .git/hooks-local need the shim renamed accordingly.
