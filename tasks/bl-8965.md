The STALL assertion in bz/src/transport/tests.rs (stall_after_first_chunk_times_out) asserts `elapsed < Duration::from_secs(5)` to prove the idle timeout fires "promptly, not hang". This is a soft wall-clock upper bound: on a heavily loaded CI runner where `recv_timeout(1s)` overshoots ~5x, it flakes (a false failure that says nothing about correctness).

The lower bound (`elapsed >= 900ms`, "waited ~idle") is the real proof and is sound. The ceiling only needs to prove "did not hang forever", which a tight 5s figure does poorly.

## Options
- (a) Widen the ceiling generously (e.g. 30s) — the test still catches a true hang (infinite) but tolerates a slow runner. Cheapest, keeps it a plain assert.
- (b) Drop the ceiling assert entirely and rely on a real hang guard: run drain_body on a worker thread and `recv_timeout` the result in the test (fail if no result within, say, 30s). Proves "not hang" without coupling to scheduler jitter. More code but deterministic intent.
- RECOMMENDED: (a) unless a hang guard is wanted for its own sake.

Note: the SLOW-BUT-LIVE test is NOT affected — its `elapsed >= 1s` is a floor backed by `thread::sleep` gaps, and it asserts the full reassembled body (coalescing-proof), so leave it.
