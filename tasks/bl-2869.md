bl-d54a (LANDED 2026-06-17) made brazen strip the Codex-rejected sampling/length params before encode. The OFFLINE side is fully covered (tests/config_strip.rs proves `strip_unsupported` clears `max_tokens`/`temperature`/`top_p` from the canonical request). What is NOT covered is the LIVE end-to-end assertion: that a `bz --provider openai-chatgpt` run CARRYING `--temperature`/`--top-p`/`--max-tokens` now returns 200 (the value normalized away) where pre-fix it 400d `{"detail":"Unsupported parameter: <field>"}`.

## Why it is worth a live case
The strip happens in `serve` on the canonical/non-`--raw` path, exactly like the `stream:true` force (src/run/serve.rs). The live OAuth fuzz/encode harnesses (bl-b72f `live_fuzz_openai.rs`, bl-f8f7 `live_encode_openai.rs`) already drive the real Codex backend via `bz`; this is one more SPEND-gated acceptance case proving the fix holds against the live service, not just the offline encoder. It also guards against a future regression where a param slips back onto the wire.

## Shape (no new infra)
Add ONE acceptance case to the existing `BRAZEN_LIVE_FUZZ_SPEND=1` set (reuse the `bz/tests/live_support/openai.rs` shared leaf + `exec.rs::run_bz`):
- argv: the working canonical body (instructions/stream/store:false) PLUS `--temperature 0.5 --top-p 0.9 --max-tokens 64` (or the same three keys in the input JSON).
- assert: exit 0 (a normal completion), NOT exit 69/`Unsupported parameter`. That is the whole contract — brazen stripped them, codex accepted the request.
- token-costing (it is a real completion), so gate behind `BRAZEN_LIVE_FUZZ_SPEND=1` like the other acceptance cases.

## Caveat (mirror bl-22d5/stream-false)
Because the strip is on the canonical path, a `--raw` probe would NOT strip (raw bypasses encode) — so this case MUST use the canonical/`--json`/`--text` path, not `--raw`. Document that inline so a later reader does not "fix" it to `--raw` and break the assertion. See [[bz-live-openai-chatgpt-testing]].

## Not in scope
The offline behavior (already done in bl-d54a). Other backends (unaffected).
