Deliverable: specs/ingress.md (new living spec) + the architecture.md §2 amendment revising the no-input-dialect decision + specs/README.md table row. Design content settled in conversation 2026-07-09/10: ingress codecs (per client dialect: request decoder + response/SSE encoder) so a harness speaking openai_chat can drive any upstream through the canonical pivot (M+N codecs, never MxN bridges). Scope wave 1 = openai_chat ingress; anthropic-messages second. Core decisions to write down: (1) adapt-or-reject ladder: representable=transform silently; zero-loss adaptation=do silently; lossy adaptation=obvious default allowed but ALWAYS an exposed knob (one policy mechanism, not a flag farm) and runtime-discoverable; no adaptation=reject at the edge in the CLIENT dialect before any round-trip. A predictable upstream 400 is a brazen bug. (2) Hard edge: carry the SPEC, not the water — pre-reject only STRUCTURAL impossibilities (wire has no slot); provider POLICY restrictions (value ranges, entitlements, model-conditional quirks) propagate as the upstream error. (3) Replay/opaque-payload cache: xdg-path store keyed by what the client echoes (tool-call ids; content hash for non-tool turns), fail-open BY DEGRADING THE KNOB (e.g. omit thinking on cache-miss replay turn), never by sending a known-invalid body; cache = fidelity optimization, absence degrades never breaks, so statelessness survives. (4) Routing: separate config surface mapping client model alias to upstream row+model. (5) Listener: opt-in, localhost-bound, optional bearer; plus one-shot stdin filter mode question. (6) Pseudo-routes (GET /v1/models from existing model cache). (7) Error masquerade: upstream errors re-encoded into client-dialect envelope+status so client retry logic works. (8) Fidelity: exact SSE shapes, tested by driving real SDKs as clients.
