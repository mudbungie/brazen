Arch-review finding (2026-07-07), confirmed by two independent audits. The openai CHAT decoder is the ONLY one of the five with no in-band error branch: chunk() (src/protocol/openai/decode/mod.rs:83-111) reads only choices[0]/finish_reason/usage; a frame 'data: {"error":{...}}' on a 200 stream produces zero events. Peers all handle it: anthropic decode/mod.rs:94, google decode/mod.rs:28-30, ollama decode/mod.rs:27-29, responses decode/mod.rs:49. Consequence: the real provider error is discarded; the run ends as generic premature-EOF Transport/69 (misdiagnosed as retryable network blip), or — worse — silently exits 0 with truncated output if a [DONE] follows the error frame.

The spec bakes in the misconception: openai-chat-mapping.md:414 asserts Chat Completions never emits in-band SSE errors on a 2xx stream. False in the ecosystem this dialect explicitly serves via one-row reuse (providers.md §2: Mistral row, and the Azure/OpenRouter/LiteLLM/vLLM class): those backends routinely emit data:{error:...} mid-200-stream. Amend the spec (the sibling google spec providers.md:290 already documents the identical hazard it fixed).

Second, related: decode sets state.terminated ONLY on [DONE] (decode/mod.rs:73-76); the finish path (decode/blocks.rs:60-65) emits Finish without terminating. Some OpenAI-compatible servers close after the finish_reason chunk without [DONE] -> clean completion reported as premature-EOF/69. Decide + spec: treating a non-null finish_reason chunk as the terminal marker matches the google/ollama field-on-chunk precedent (providers.md §4.4/§5.5) and loses only truncation detection in the finish->[DONE] window (no data there). Fixture for both: mid-stream error frame (then with and without trailing [DONE]) + finish-then-EOF-no-DONE.

Scope: src/protocol/openai/decode/*, openai-chat-mapping.md §3.x + §6 note, golden fixtures, CHANGELOG. Error-kind mapping for the in-band case follows CR-10 (no governing status -> kind from body: rate-limit-ish -> Provider{429}, server-ish -> Provider{500-class}, else Transport — mirror anthropic's mid-stream table).
