## Symptom
`bz --provider <p> --raw` (provider-native passthrough) ALWAYS fails: exit 69, empty stdout, empty stderr — the request never leaves the machine. Reproduced live against Ollama AND a local capture server (server received nothing).

## Root cause
`WireRequest::raw(body)` (src/protocol/mod.rs:52-54) is `WireRequest::new(String::new(), body)` — URL is the empty string. The canonical path gets its URL from `proto.encode` (which builds `{base_url}/responses` etc.), but the raw path in src/run/serve.rs:87 (`Input::Raw(bytes) => WireRequest::raw(bytes)`) BYPASSES encode and nothing ever fills `wire.url` from `cfg.provider.base_url` + the protocol path. So the transport POSTs to "" → connect error → exit 69. RawSink swallows the in-band Event::Error (it only writes Event::Raw), hence empty output.

## Why offline tests miss it
MockTransport ignores `wire.url`, so every offline raw test passes with an empty URL. Only a real transport reveals it — exactly the gap the live conformance suite (bl-04dc) is for.

## Fix sketch
The raw path needs the same target the encoded path computes: `{base_url}` + the protocol's request path. Either have the raw path ask the protocol for its path (a `Protocol::path()` or reuse the encoder's URL construction) and set `wire.url`, or thread base_url+path into `WireRequest::raw`. Single source of truth: the protocol already owns the path string in encode — don't duplicate it.

## Repro
echo '{"model":"M","messages":[{"role":"user","content":"hi"}],"stream":false}' | bz --provider ollama --model M --raw  # exit 69, nothing sent

Found by bl-04dc (live conformance suite).
