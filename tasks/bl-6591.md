Validated the automation end-to-end: green CI -> workflow_run -> publish job RUNS (gate works); cancelled CI -> publish SKIPPED (build-gate works). But the publish job itself errors before reaching the token: 'cannot determine current branch ... HEAD does not point to a branch'. Cause: release-plz.yml publish job checks out 'ref: github.event.workflow_run.head_sha' -> detached HEAD; release-plz needs to be ON a branch. Fix: 'ref: github.event.workflow_run.head_branch || github.ref_name' (='main' for both workflow_run and workflow_dispatch) so it checks out the main branch, not a detached SHA. After this, the publish job should reach the actual publish step and fail only on the (unset) CARGO_REGISTRY_TOKEN — the correct armed-waiting state. Close gate: YAML parses; make check green; verify next workflow_run publish job's failure is the token, not the branch.
