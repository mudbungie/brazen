## Context

One-way-door review (2026-06-27). A worry was raised that "one credential file per
provider" (`<provider>.json`) walls off multiple accounts per provider. It does NOT: the
cred-store key is the provider ROW NAME (`CredStore::get/put` keyed by name), and config
allows N rows. So `anthropic` and `anthropic-work` resolve to separate `<name>.json`
files — multi-account, multiple keys, and key-rotation already work TODAY by naming rows.

## Decision (owner): good — lock it as a supported pattern so a future refactor doesn't
break it.

## Deliverable

Document in auth.md / config.md (living doc): multiple provider rows with distinct names,
each resolving to its own credential file, is the SUPPORTED mechanism for multiple accounts
/ multiple keys / rotation per provider. State it as an interface guarantee — the cred-store
key is the row name; do not collapse it to a single per-vendor file.

Type: **DOC** (records a supported-interface guarantee; small).
