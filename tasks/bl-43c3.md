title = "OAuth RNG silently returns all-zeros token on /dev/urandom failure (predictable PKCE verifier + CSRF state)"

bz/src/native/rng.rs fill_random (Unix) discards the read/open error with `if let Ok`/`let _ =`; on failure buf stays [0u8;32], so random_token() returns a constant base64url token used as the PKCE verifier and CSRF state. random_token returns a bare String so it cannot fail loudly. Contradicts the auth spec's 'never a silent fallback'. The bz shim is coverage-excluded (Makefile --ignore-filename-regex 'bz/'), so no test guards it.

Fix: make RNG failure fatal on the (interactive, control-plane) login path rather than emitting a predictable token. Open AND read_exact must both succeed or abort. Non-Unix weak time-seed fallback (auth §5.2) is a separate documented limitation, out of scope here.
