Implements the owner decision frozen in architecture.md §5.5 / §13.7 / §5.10 (design: bl-a614). Serialized after bl-730f (both rewrite cli.rs::parse_args; --needs avoids the conflict).

RULE (clear skies / brick walls): option parsing stops at the FIRST token that is not an option or an option-value; everything from there through EOF is the prompt, operand tokens joined by a single space. No token after the prompt starts is an option.

cli.rs::parse_args:
- On the first non-option token (or after --): collect ALL remaining argv to EOF, join with ' ', set flags.prompt. Stop the flag loop.
- Drop set_prompt's 'only one positional prompt is allowed' error (silent join is now the contract).
- First token starting with '-' that isn't '--' and isn't a recognized flag -> unknown-flag usage error 64 (PRESERVED: leading-flag typo detection).
- '--' still ends options; everything after it (to EOF, joined) is the prompt; empty after -- => no positional (falls to stdin/bare path).

CONSEQUENCES (documented, intended):
- bz a b c            -> prompt 'a b c'
- bz --json "q"       -> JSON + prompt 'q'
- bz "q" --json       -> prompt 'q --json' (output stays text; post-prompt flag is inert)
- bz -- --weird       -> prompt '--weird'
- bz -x               -> unknown flag -> 64
- positional present  -> stdin NOT read, pipe silently ignored (head idiom); NO XOR error (already true in read_request; §5.5 doc fixed in bl-a614)

main.rs: reconcile the stale 'positional-prompt drain / XOR error' comment wording (the empty-reader swap is for the NO-positional bare-tty case only).

TESTS (cli.rs table + run integration): each consequence above; multi-word join; -- tail join; leading-dash -> -- escape; first-token unknown flag 64; flags-before-prompt selects mode while same flag after prompt is prompt text.

GATE: make check (100% cov + 300-line cap).
