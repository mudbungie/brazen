## Observation (during bl-1156 close, 2026-06-27)

The offline coverage gate `make cov` — i.e.

    cargo llvm-cov --fail-under-lines 100 --ignore-filename-regex 'src/(main\.rs|native)'

printed `TOTAL ... 98.52%` line coverage with **104 missed src lines** yet **exited 0**, and the
full `make check` (the pre-commit gate) passed, delivering b13adec. A `--fail-under-lines 100`
threshold at 98.52% is *expected to fail* (non-zero exit), so the gate's pass is unexplained.

Missed lines were spread across real src files, e.g.: `store.rs` 97.67% (2), `run/respond.rs`
98.92% (3), `run/models.rs` 97.81% (4), `testing/login.rs` 96.08% (2), `testing/transport.rs`
97.56% (2), `protocol/openai/encode/messages.rs` 98.41% (3), plus several `protocol/*/decode|encode`
files at 99.x%. The TOTAL was a **stable** 98.52% / 104 across two separate runs (`make check` and a
standalone `make cov`), so it was not transient flapping.

## Caveat — many agents were live

A large parallel `bl` mass-execution was closing tasks concurrently during this run. Per the
`bl multilane gotchas` memory, concurrent closes can spuriously perturb a parallel `cargo test`.
So contaminated profdata is a live hypothesis and the repro MUST be on a QUIET repo.

## Hypotheses to check

1. **Live-test-only lines.** The 104 lines are exercised only by `#[ignore]`'d live tests (skipped
   offline), so they read uncovered — but then `--fail-under-lines 100` should still FAIL on the
   displayed number. If it doesn't, the threshold is computed on a DIFFERENT metric than the printed
   src TOTAL (region/function? a broader `--ignore-filename-regex`? a `[package.metadata.llvm-cov]` /
   `llvm-cov.toml` / `.cargo/config` override?). Find what the flag actually checks against.
2. **Parallel-agent contamination** of the profraw/profdata merge produced an inconsistent
   report+exit. Re-run quiet to rule in/out.
3. **cargo-llvm-cov version quirk** in how `--fail-under-lines` is evaluated vs. the printed table.

## Deliverable

On a clean `main` with NO other agents running:

- Repro: `make cov; echo $?` and `make check; echo $?`; inspect whether the exit reflects the
  displayed TOTAL. Capture the cargo-llvm-cov version.
- Determine which hypothesis holds.
- If the gate genuinely passes below 100%, **fix it** so the offline gate enforces the AGENTS.md
  invariant ("100% line coverage via `cargo llvm-cov --fail-under-lines 100`"). If the displayed
  number legitimately differs from the threshold metric (e.g. live-only lines excluded by design),
  **document that** in AGENTS.md / Makefile so the next reader is not alarmed — reconcile the living
  docs either way.
