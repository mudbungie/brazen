Design task. Deliverable = amend specs/model-discovery.md (rework §1/§2/§4/§5 — the probe is DISSOLVED into an unconditional cache lookup) + a CR to architecture.md §2. Living doc, edited like code (AGENTS.md design-first). Implementation is a SEPARATE later ball.

CORE DECISION: brazen NEVER lists automatically. `bz list-models` is the ONLY thing that hits the models endpoint and the ONLY writer of the cache; the generation path is READ-ONLY against the cache. Each invocation stays POSIX-clean (one network action, or zero) and arch §2 holds verbatim ("Not an agent ... no retry/backoff ... the caller orchestrates"). The cache-rule violation is ACCEPTED (a regenerable XDG file); auto-spawn / auto-retry is REJECTED.

THE CACHE LOOKUP IS UNCONDITIONAL. There is NO skip-cache / "user-verbatim" path. Once the provider is resolved, EVERY model string — full id or partial — is a SEED run through the one resolver against that provider cache. Verbatim is the FALLBACK when the cache cannot resolve the seed, not a path the caller selects. This DISSOLVES the `probe` bool: `needs_probe` / `row_owns`-for-probe / `ResolvedConfig.probe` all GO (supersedes bl-67e2). `model_prefixes` survives but ONLY for ROUTING (which provider owns a full id), never an owned-vs-probe decision.

THE ONE RESOLVER (select_model over the loaded cache list):
  - exact id match in cache -> that id
  - else FIRST substring match (list order) -> that id ("opus" -> claude-opus-4-...; a full id substring-matches its own line)
  - else -> the SEED VERBATIM (attempt literally)
  - empty seed ("" = absent model) -> the default (models[0], or the flagged default)
A cache MISS is not a special branch: it is select_model over an EMPTY list (AGENTS.md empty-input dissolve; cache-absent == cache-present-but-empty). Load the cache into a Vec<Model> (empty if no file), run the one resolver. Every non-empty seed then falls through to verbatim; only the empty seed has nothing to fall through to.
  -> SELF-HEALING: a full id of a newly-added model absent from a STALE cache gets no match -> attempted verbatim -> SUCCEEDS. A partial with no match -> verbatim -> provider 404 -> user re-lists.

select_model CONTRACT CHANGE (vs the landed bl-76e3 impl): a non-empty seed that matches nothing now RETURNS THE SEED VERBATIM, no longer Config 78 NoMatch. The NoMatch-78 + stale-hint from the prior draft is GONE — no-match self-heals to a literal attempt.

THE LONE PRE-EMPTIVE FAIL: empty seed ("") AND empty cache -> nothing to send and no list to default from -> FAIL EX_NOINPUT (66) + message: specify a model or run `bz list-models`. (Was: every cold miss returned 66; now only this corner.) A configured default model makes the seed non-empty, so this only bites a truly bare invocation against an unlisted provider.

404 ON THE GENERATION REQUEST — disambiguated by PROVENANCE (carry the came-from-cache-match fact, AGENTS.md "carry, dont reconstruct"):
  - wire id came from a cache MATCH (exact/substring) that 404s -> the listed entry was deprecated since listing -> exit 69 + hint: cache may be stale, re-run `bz list-models`. We KNOW it was on the list.
  - wire id came from the VERBATIM fallback (no cache match / empty cache) that 404s -> exit 69 + hint: no cache match; run `bz list-models` to enable fuzzy matching / refresh. Covers cold-cache and stale-missing.
Both exit 69; only the hint text differs, driven by the one provenance bool.

WHAT THIS DELETES vs prior drafts / in-flight impl:
- `probe` / `needs_probe` / `ResolvedConfig.probe` — DISSOLVED (cache lookup is unconditional). Supersedes bl-67e2.
- The auto-probe (§5.2 `fetch_models` in serve) — REMOVED; generation reads the cache. Supersedes that part of bl-1189 (the `list-models` verb + `Model`/`decode_models` + the select_model SKELETON are kept; select_model SEMANTICS change per above).
- The `--fail-on-404` flag — GONE.
- The front-door SUBPROCESS — GONE (generation never lists). "Front door" survives only as: list-models is the sole cache writer.
- The infinite loop — GONE by construction.
- The NoMatch-78 fail and the every-cold-miss-66 — GONE (replaced by verbatim fallback; 66 shrinks to the empty-seed-no-cache corner).

COST MODEL: full id + warm cache = 1 round-trip; full id + cold cache = 1 (verbatim, self-heals); partial + warm cache = 1; partial + cold cache = 1 (verbatim -> likely 404 -> user lists). `bz list-models` = 1, separate invocation. NO path does two round-trips.

CACHE STORAGE: key = provider row name (cfg.provider.name, same key as CredStore store_key). Location = $XDG_CACHE_HOME/brazen/models/<provider>.json (REGENERABLE -> CACHE_HOME, not creds/STATE). Format = the {"models":[{id,default}]} shape `list-models --json` already emits. No TTL (sole invalidation is the user re-running list-models). Atomic write (temp + rename) for concurrent `bz`.

CARRIED FACT: a provenance bool "wire id came from a cache match (vs verbatim fallback)" threaded from the resolver into the 404 handler so the hint can branch.

CR to architecture.md §2 (changes FIRST, §7 CR discipline): add a regenerable model-list cache as a sanctioned XDG exception alongside config + creds. §2 "no retry/backoff, caller orchestrates" is HONORED, not bent: a wrapper maps a 404/66 -> `bz list-models` -> retry; brazen exposes the condition and never acts.
