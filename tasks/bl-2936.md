## Context

One-way-door review (2026-06-27). Pre-ship audit. Current deps (Cargo.toml):
- lib/shared: `serde`, `serde_json`, `toml` 0.8, `sha2`, `base64`
- impure shim only: `ureq` 3 (rustls, `default-features=false`), `libc` (unix)
- dev: `tempfile`

## Decision (owner): reduce the dependency surface as much as possible.

## Deliverable

Audit every dependency and its enabled feature set; remove or replace what isn't pulling
its weight. Specifics:
- `toml` 0.8 — needed only to parse config + embedded defaults.toml; confirm minimal
  features (no `preserve_order` / `serde-spanned` unless used).
- `sha2` + `base64` — used only for PKCE S256 (one challenge). Confirm both are still worth
  it vs. a smaller path; trim `base64` to the one engine/alphabet used.
- `ureq` — already `default-features=false, features=["rustls"]`; verify no transitive
  bloat (webpki-roots bundling), no unused features.
- `serde_json` — confirm no unneeded features (`arbitrary_precision`, etc.).
- Run `cargo tree` / `cargo machete` / `cargo udeps` to catch unused or duplicated
  transitive crates. Keep the no-C-deps, single-binary, musl/cross-compile story intact
  (architecture.md §10).

Goal: smallest dep + feature footprint that still builds the `bz` binary on the full target
matrix.
