## Context

`CanonicalError::retryable()` (`src/canonical/error.rs`) is the computed query for
"is this error retryable" — `matches!(kind, Transport) || matches!(kind, Provider{status}
if status == 429 || status >= 500)`. The §3.3/§3.5 single-source-of-truth ledger lists
`retryable` as a **computed query** (a method, not a stored field, so it cannot drift).

Surfaced during the exit-code-granularity review (bl-3ce8, 2026-06-27): `retryable()` has
**no production caller**. It is exercised only by tests (`tests/canonical_error.rs`,
`tests/openai_decode_errors.rs`, `tests/google_decode_errors.rs`). The fact reaches a
`bz` consumer on the wire via the serialized `kind` (`Provider{status}`) in `--json`, from
which the consumer recomputes retryability itself. So the method is, today, dead weight in
the binary.

## Decision (owner): EVALUATE — do NOT remove regardless.

This is a one-way-door-adjacent judgment call, not a directive to delete. Removal must be
weighed and the outcome recorded (even if the outcome is "keep, as intended public API").

## Considerations

KEEP arguments:
- `brazen` is a **library** + the `bz` shim. `retryable()` is plausibly a public API for
  *embedders* (who call the lib, not the CLI), even though no in-tree binary needs it.
- It is the single **documented home** of the retryable predicate. Delete it and the only
  home of "what counts as retryable" becomes prose plus each embedder re-implementing
  `429 || >=500` — exactly the drift the §3.5 ledger ("two reps would drift") guards against.
- "If it can't be tested it isn't built" is satisfied — it IS tested.

REMOVE arguments:
- No production caller; YAGNI. The wire already carries `Provider{status}`, so the fact is
  not lost — a consumer (CLI or embedder) can compute it from the status.
- A public API surface with zero in-tree consumers is unproven; "tested by tests that exist
  only to test it" is close to gaming coverage on a method nothing uses.

## Deliverable

A short design decision (record the outcome even if "no change").
- If **KEEP**: document it as a deliberate public-API-for-embedders surface (a one-liner in
  §3.3 or the §3.5 ledger) so it is not re-flagged as dead code next review.
- If **REMOVE**: delete the method + the tests that exist only to exercise it, and reconcile
  every `retryable` reference in the §3.3/§3.5 ledger and the sibling specs
  (`anthropic-messages.md`, `openai-chat-mapping.md`, `providers.md`, `model-discovery.md`)
  that lean on it as the named home of the predicate.

Type: **DESIGN** (eval + decision).
