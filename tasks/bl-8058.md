`bz login` exists + an XDG CredStore, but a run still needs --api-key/BRAZEN_API_KEY; nothing reads the store implicitly for a provider, and brazen doesn't know about the ambient Claude Code credential (~/.claude/.credentials.json). Make a resolved provider pull its stored cred automatically (login once → run with no key), and consider discovering the Claude Code OAuth credential as a zero-setup source. Removes `jq … --api-key` from the one-liner.
