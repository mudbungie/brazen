The idle-read timeout (bl-3394) lives in `bz/src/transport.rs` (`IdleChunkReader`), which is coverage-excluded (the impure shim), so its behavior is reasoned-but-unexercised. Add a real integration test that drives `HttpTransport::send` against a localhost server controlling response timing.

## Scope
- ONE generic stall server suffices, NOT one per protocol: the timeouts act on the raw HTTP byte stream below the protocol layer, so encode/decode is irrelevant. Drive `HttpTransport::send` directly with a hand-built `WireRequest` (set `timeouts.idle`), point `url` at a `TcpListener` bound on `127.0.0.1:0` in a test thread, drain the body iterator, measure.

## The two assertions that prove the feature
1. STALL: server writes 200 + headers + one body chunk, then sleeps forever. The body iterator must yield a `TimedOut` error after ~idle (exit-69-class), NOT hang. Proves it fires.
2. SLOW-BUT-LIVE: server writes a chunk every gap-slightly-under-idle for several seconds, then ends cleanly. Must complete with NO timeout. Proves the per-chunk reset — total stream length is uncapped (a naive total-body cap would wrongly truncate this). This is the load-bearing test.

## Decisions
- Granularity: config is whole seconds (`u64`), so the finest idle is 1s. RECOMMENDED (a): test with idle=1s (stall returns in ~[1.0, 2.0]s; live stream uses ~300ms gaps over ~2s). No code change. Alternative (b): switch the timeout unit to fractional seconds (`f64`/`from_secs_f64`) or millis for sub-second precision + faster tests — only do this if sub-second timeouts are wanted as a real feature.
- Avoid tight upper-bound timing asserts (CI-flaky on loaded runners). Assert lower-bound "waited ~idle" + a generous ceiling "did not hang" (e.g. < 5x idle). The slow-but-live completion is the deterministic one.
- `bz` is a binary crate (not importable from `bz/tests/`), so the test is a `#[cfg(test)]` module in its OWN file under `bz/src/` to respect the 300-line cap. Does NOT move the coverage metric (`bz/` is excluded by design) — value is behavioral verification.
