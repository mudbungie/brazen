# Rework prompt caching to ZERO interface: delete the bl-d25b `req.cache` surface; the Anthropic encoder places `cache_control` automatically

## Ruling (user, 2026-07-01 ‚Äî supersedes bl-d25b's interface; do not re-litigate)
bl-d25b (shipped `40b795d`) put `cache: Vec<CacheBreakpoint>` + `CacheAnchor{Tools,System,Message{index}}` + `CacheTtl{FiveMin,OneHour}` on `CanonicalRequest` ‚Äî a MECHANISM interface: the caller reasons about anchors, TTLs, and the 4-cap, and the canonical surface names a provider's dialect in its own doc comment. The design goal of brazen is that the harness knows only brazen, never the provider. Caching is therefore now brazen-OWNED POLICY with ZERO canonical surface: no field, no flag, no config key. The encoder decides placement from the request's own shape; the caller observes the outcome through the existing `Usage.cache_read_tokens`/`cache_write_tokens` (untouched). The type break is sanctioned ‚Äî 0.0.x is draft.

## 1. Delete the shipped surface

- `src/canonical/mod.rs` + `src/lib.rs`: remove the three re-exports. `tests/interface_parity.rs` derives the closure mechanically ‚Äî confirm it passes with the shrunken surface.

- Breaking-input note: a piped request still carrying the old `"cache": [...]` key now falls into `extra` (fail-open) and rides to the wire, where the provider rejects the unknown key. Acceptable pre-0.1; CHANGELOG records it. Do NOT add a strip or a compat shim.

## 2. The auto policy (rewrite `src/protocol/anthropic/encode/cache.rs`, same call site)
`apply(body, req)` keeps its signature and call position (after typed field insertion, BEFORE the `extra` fold ‚Äî a policy marker still wins over a raw `extra` `cache_control`, the existing ¬ß2.1.1 rule). It no longer reads any request field; it computes marks from the request's shape:

1. **Head mark ‚Äî always.** `cache_control: {"type":"ephemeral"}` on the LAST `system` wire block; if `system` is empty/absent, on the LAST `tools` object; if both are empty, no mark (empty-set rule). Sub-minimum prefixes (1024/4096 tokens by model) are Anthropic's documented SILENT NO-OP ‚Äî never brazen's to police, so an unconditional head mark costs a one-shot nothing (small head) or at most one 25% write premium (big head, never recurs).
2. **Rolling conversation mark ‚Äî when the request is an ongoing conversation.** Trigger: at least one `assistant` message at an index BEFORE the last message. (A lone TRAILING assistant is a prefill one-shot ‚Äî no trigger: a prefill is extended by the next turn's completion, so its blocks mutate and must not anchor a cache.) Placement: the last ELIGIBLE wire block of the LAST NON-ASSISTANT message; if that message projects to 0 eligible blocks, walk back to an earlier message. Reuse the existing `wire_pos` machinery (the System-hoist skip: wire position = count of non-System messages before the index).
3. **One intermediate mark** ~20 eligible wire blocks before the rolling mark, only when that many blocks exist between the head and the rolling mark ‚Äî this keeps the PREVIOUS turn's write point inside Anthropic's 20-block cache-hit lookback even when a turn's delta is large. Total marks ‚â§4 by construction (head + intermediate + rolling = 3); the provider cap can never be hit, so no validation error exists on this path at all.
4. **Eligibility:** `cache_control` is invalid on `thinking`/`redacted_thinking` blocks ‚Äî when the natural target is one, step back to the previous eligible block.
5. **TTL: always omitted** (the 5m default; it renews on every cache read, so a steady loop stays warm indefinitely). 1h only wins across idle gaps a stateless adapter cannot see. Not exposed anywhere.

Placement is MECHANISM ‚Äî tunable in later balls with no interface change. All other protocol encoders: nothing to do (their providers cache prefixes automatically; the field they used to ignore no longer exists).

## 4. Recorded non-goals (write them into the spec)
- **XDG prefix-hash journal** (observe cross-invocation recurrence for precise placement): the upgrade path if the stateless heuristic's misses ever matter in practice.
