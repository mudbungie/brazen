Two confirmed decode bugs in src/protocol/google_genai/decode/, both worth fixing BEFORE the v=1 wire-vocab freeze (a later thought->Thinking correction is a SEMANTIC change of an existing delta, forcing an EVENT_SCHEMA_VERSION bump).

BUG 1 — thought leaks as answer text (providers.md §4.4). src/protocol/google_genai/decode/blocks.rs:17-24 part_events() reads part["text"] and emits Delta::TextDelta, with NO branch for the Gemini thought flag. A thinking part is {"text":"...reasoning...","thought":true} (emitted when the caller opts in via thinkingConfig.includeThoughts, carried through req extra). So private chain-of-thought is emitted as the assistant's answer, AND it is cross-provider-INCONSISTENT: Anthropic and openai_responses correctly emit a canonical Thinking block. Fix: when part["thought"].as_bool()==Some(true), route the text into a Thinking content block (open/extend a Thinking block + ThinkingDelta), mirroring how src/protocol/anthropic/decode and src/protocol/openai_responses/decode open Thinking. Reuse the synth helpers (protocol::synth) — add a thinking analog of open_text if needed.

BUG 2 — prompt-level safety block -> premature-EOF/69 instead of Refusal/exit 0 (providers.md §4.4). src/protocol/google_genai/decode/mod.rs:59-70 chunk() reads only v["candidates"][0]. A prompt-level block returns {"promptFeedback":{"blockReason":"SAFETY"}} with NO candidates and no finishReason, so it produces a spurious MessageStart then falls through with no terminal event -> the stream ends without Finish -> decoded as premature upstream EOF -> ErrorKind::Transport/exit 69 (retryable=true). It should decode to Finish{Refusal}/exit 0 and set state.terminated, matching the spec's deterministic-block semantics. Fix: when a chunk has no candidates but carries promptFeedback.blockReason, emit Finish{Refusal} (+ ContentStop drain if any open) and flip terminated.

CONSTRAINTS: make check is the close gate — 100% LINE coverage + clippy -D warnings + fmt + 300-line/file cap. Every new line MUST be exercised: add a google thinking SSE fixture (thought:true part) and a google prompt-block fixture (promptFeedback, no candidates), wired into the existing google decode tests (see src/tests/google_decode_errors.rs, src/tests/google_fixtures.rs, tests/fixtures/google_*). Reconcile providers.md §4.4 if the spec wording needs it (the code is the deviation to fix, not the spec — but if the spec is silent, add the CR note).
