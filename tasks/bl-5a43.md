OWNER DECISION (Q2): SCOPE it to the anthropic row (not drop, not document). Confirmed bug: src/config/env.rs:43-46 maps ANTHROPIC_API_KEY into the UNIVERSAL api_key field -> becomes AuthCtx.inline_key -> resolved_secret() returns it FIRST for ANY provider (src/auth/mod.rs:218), BEFORE the store. Two harms: (1) a stray ANTHROPIC_API_KEY in any dev shell is transmitted to OpenAI/Google/etc; (2) it SHADOWS a correctly bz-login'd/stored cred. (BRAZEN_API_KEY is the brazen-native explicit signal and STAYS provider-agnostic; only the vendor-conventional ANTHROPIC_API_KEY alias must be scoped.)

HARD CONSTRAINTS:
- NO 'provider == "anthropic"' vendor branch anywhere — that violates the §4.4 'dispatch with no match-on-provider' north star. The scoping MUST be data-driven so it generalizes to any row and is severable (AGENTS.md: deleting the row datum deletes the capability).
- The provider-native env key must NOT shadow a stored/bz-login'd cred: it belongs at AMBIENT/low precedence (read on a store MISS), below --api-key / BRAZEN_API_KEY, scoped to the resolved row.

TWO CANDIDATE APPROACHES — pick whichever is cleaner against the actual code:
  (A) Route it through the EXISTING per-row ambient mechanism (src/store.rs AmbientSpec/AmbientFormat, auth §5.5 — already provider-scoped, store-miss, severable). Extend the closed AmbientFormat with an env-var source (the native XdgCredStore.discover reads the env var; the lib stays pure), give the anthropic row an ambient block naming ANTHROPIC_API_KEY in data/defaults.toml, and REMOVE the ANTHROPIC_API_KEY line from env.rs. This reuses the designed severability seam.
  (B) Add a per-row key_env: Option<String> (config/provider.rs, additive/schema-safe), have resolution read the RESOLVED row's key_env from the EnvSnapshot as a low-precedence credential, remove the env.rs hardcode. Data-driven, no vendor branch.
Evaluate both; prefer the one that adds least mechanism and best fits the existing seams.

SCOPE: src/config/env.rs, src/store.rs and/or src/config/provider.rs, src/config/resolve.rs, src/auth/mod.rs (precedence), data/defaults.toml, config.md + auth.md §5.5. Do NOT touch cli.rs/main.rs/lib.rs/canonical/*. 0.0.x cut (doors open) so an additive schema change is fine. make check is the close gate (100% line cov + clippy + fmt + 300-line); add tests proving ANTHROPIC_API_KEY reaches ONLY anthropic and does NOT shadow a stored cred for other providers.
